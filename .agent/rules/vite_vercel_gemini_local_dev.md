# Setup Vite + Vercel Serverless API + Gemini SDK

This workflow documents common issues and their solutions when building a React/Vue SPA using Vite, deploying it to Vercel, and utilizing Vercel Serverless Functions (`/api/*`) for backend tasks like communicating with the Gemini AI SDK.

## Issue 1: Vite Intercepting `/api` Routes Locally
**Problem:** 
When running `npm run dev` or `vercel dev`, Vite attempts to parse the `index.html` file when the frontend makes a fetch request to `/api/chat`. This results in a "Failed to parse source for import analysis" error in the console.

**Solution:**
Do not rely purely on `vercel.json` rewrites for local development, and avoid complex proxy loops. Instead, inject a custom middleware plugin directly into `vite.config.js` to natively execute your Vercel API endpoints during Vite development.

```javascript
// vite.config.js
import { defineConfig, loadEnv } from 'vite';
import react from '@vitejs/plugin-react';

export default defineConfig(({ mode }) => {
  const env = loadEnv(mode, process.cwd(), '');
  
  return {
    plugins: [
      react(),
      {
        name: 'mock-vercel-api',
        configureServer(server) {
          server.middlewares.use('/api/chat', async (req, res, next) => {
            if (req.method === 'POST') {
               try {
                 // 1. Inject local env variables so your serverless function can read them
                 process.env.GEMINI_API_KEY = env.GEMINI_API_KEY;
                 
                 // 2. Dynamically import your Vercel function
                 const { default: handler } = await import('./api/chat.js');
                 
                 // 3. Read request body
                 let body = '';
                 req.on('data', chunk => { body += chunk.toString(); });
                 req.on('end', async () => {
                   req.body = body ? JSON.parse(body) : {};
                   
                   // 4. Mock Vercel specific Response methods
                   const vercelRes = {
                     status: (code) => { res.statusCode = code; return vercelRes; },
                     json: (data) => {
                       res.setHeader('Content-Type', 'application/json');
                       res.end(JSON.stringify(data));
                     }
                   };
                   
                   await handler(req, vercelRes);
                 });
               } catch(err) {
                 console.error(err);
                 res.statusCode = 500;
                 res.end('Error loading API handler');
               }
            } else {
               next();
            }
          });
        }
      }
    ],
  };
});
```

## Issue 2: Gemini API `GenerativeAIError` (Role history constraint)
**Problem:**
The Gemini API will crash with `Internal Server Error (500)` if the chat `history` array starts with a message from the `model`. It strictly requires the first message in the context history loop to originate from the `user`.

**Solution:**
In your serverless API route (`api/chat.js`), format the history from the frontend and actively `shift()` out any initial welcome messages generated by the bot until a user message is the true start.

```javascript
    // Format history loop
    // ...

    // Ensure the first message in history is from 'user'
    // Gemini API requires the history to start with a user message to provide conversational scaffolding.
    while(formattedHistory.length > 0 && formattedHistory[0].role !== 'user') {
        formattedHistory.shift();
    }
    
    // Split the last message to act as the current prompt
    const historyWithoutLatest = formattedHistory.slice(0, -1);

    // ... (Initialize Chat Session with historyWithoutLatest)
```

## Running the Application
Always run:
`npm run dev`

With the Vite plugin above, both the Vite frontend and your Node.js Vercel backend (`api/`) will elegantly run on the same local port without conflicts.
